# 架构说明

`zephyr-mail` 采用“命令层 + 配置层 + 协议服务层 + 输出层”的分层结构。

## 1. 数据流

1. `cmd/zephyr-mail/root.go` 读取配置并注册子命令
2. `internal/cli` 解析 flags 和位置参数
3. `internal/config` 从 `.env` 和环境变量加载配置
4. `internal/imap` 或 `internal/smtp` 执行业务操作
5. `internal/output` 负责 JSON 和错误输出

## 2. 关键边界

### 命令层

只做输入校验、默认值处理和调用，不直接处理协议细节。

### 配置层

负责把环境变量转换成结构化配置，并做最小必要校验。

### IMAP 层

处理：

- 邮件列表查询
- 按 UID 获取邮件
- 附件信息检查
- 标记已读/未读
- 邮箱列表

### SMTP 层

处理：

- 邮件发送
- 连通性测试
- 发件人、正文和附件的解析与组装

### 输出层

统一处理：

- JSON pretty print
- 错误前缀 `Error: `

## 3. 设计取舍

- 选择 Cobra 是为了统一子命令与 flag 管理
- 选择 JSON 输出是为了便于脚本和自动化消费
- 将 IMAP/SMTP 拆成独立包，是为了避免协议逻辑和 CLI 绑定过紧

## 4. 扩展建议

- 新增命令时，优先在 `internal/cli` 增加新文件，再补服务层能力
- 新增配置项时，优先改 `internal/config/env.go` 和 `docs/配置说明.md`
- 新增输出字段时，注意同步 `docs/命令参考.md`
