# 开发说明

本项目是一个 Go CLI 工具，入口和实现分层明确，适合按命令逐个维护。

## 1. 目录结构

- `cmd/zephyr-mail`：程序入口和根命令
- `internal/cli`：Cobra 子命令与参数解析
- `internal/config`：`.env` 加载与配置校验
- `internal/imap`：IMAP 客户端、查询、邮件解析和操作
- `internal/smtp`：SMTP 发送和连通性测试
- `internal/output`：统一 JSON 和错误输出
- `internal/common`：通用错误处理和辅助函数

## 2. 常用命令

构建：

```bash
go build -o zephyr-mail ./cmd/zephyr-mail
```

测试：

```bash
go test ./...
```

## 3. 开发约定

- 命令层只负责参数校验和调用服务
- 服务层负责协议交互和结果组装
- 错误统一输出到 stderr，格式为 `Error: <message>`
- 成功输出尽量保持 JSON
- 资源释放使用 `defer`

## 4. 配置与调试

- 本地调试优先放置 `.env`
- IMAP 相关错误通常是账号、密码、证书或邮箱目录问题
- SMTP 相关错误通常是服务器、端口、发件人或认证问题

## 5. 提交前检查

建议至少执行：

```bash
go test ./...
go build -o zephyr-mail ./cmd/zephyr-mail
```
