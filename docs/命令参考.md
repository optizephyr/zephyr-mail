# 命令参考

下面按命令说明当前可用参数和行为。

## 全局说明

- 默认邮箱目录来自 `IMAP_MAILBOX`，若未配置则为 `INBOX`
- 出错时统一向 stderr 输出 `Error: ...`
- 成功时大多输出 JSON
- `check` / `search` / `fetch` 的邮件对象使用 `Text` 作为正文输出字段，不再包含 `HTML`
- 当邮件只有 HTML 正文时，会先去除 HTML 标签后写入 `Text`

## `check`

```bash
./zephyr-mail check [--mailbox <name>] [--limit <n>] [--recent <24h>] [--unseen=true|false]
```

参数：

- `--mailbox`：指定邮箱目录
- `--limit`：最多返回多少封，默认 `10`
- `--recent`：最近时间窗口，例如 `2h`、`24h`
- `--unseen`：仅在显式为 `true` 时生效；裸写 `--unseen` 当前会被当成 `false`

## `search`

```bash
./zephyr-mail search [--mailbox <name>] [--limit <n>] [--unseen] [--seen] [--flagged] [--answered] [--from <addr>] [--to <addr>] [--subject <text>] [--recent <24h>] [--since <date>] [--before <date>] [--uid <n|range>]
```

参数：

- `--limit`：默认 `100`
- `--unseen` / `--seen` / `--flagged` / `--answered`：布尔开关
- `--from` / `--to` / `--subject`：按内容过滤
- `--recent`：相对时间窗口
- `--since` / `--before`：支持 `02-Jan-2006`、`2006-01-02`、`2006-01-02T15:04:05Z`
- `--uid`：单个 UID 或 UID 范围

## `fetch`

```bash
./zephyr-mail fetch <uid> [--mailbox <name>]
```

- `<uid>` 必填
- 返回邮件详情 JSON

## `download`

```bash
./zephyr-mail download <uid> [--mailbox <name>] [--dir <path>] [--file <filename>]
```

- `<uid>` 必填
- `--dir` 默认 `.`
- `--file` 可用于指定要匹配的附件名
- 若未找到指定文件，会返回可用附件名列表

## `mark-read`

```bash
./zephyr-mail mark-read <uid> [uid2...]
```

- `<uid>` 至少一个
- `--mailbox` 可指定邮箱目录

## `mark-unread`

```bash
./zephyr-mail mark-unread <uid> [uid2...]
```

- `<uid>` 至少一个
- `--mailbox` 可指定邮箱目录

## `list-mailboxes`

```bash
./zephyr-mail list-mailboxes
```

- 列出服务器上的邮箱目录

## `send`

```bash
./zephyr-mail send --to <email> --subject <text>
```

参数：

- `--from`：覆盖发件人
- `--to`：收件人，必填
- `--cc`：抄送
- `--bcc`：密送
- `--subject`：主题
- `--subject-file`：从文件读取主题
- `--body`：纯文本正文
- `--body-file`：从文件读取正文
- `--html-file`：从文件读取 HTML 正文
- `--html`：将 `--body-file` 当作 HTML 处理
- `--attach`：逗号分隔的附件路径

优先级：

- `--subject-file` 覆盖 `--subject`
- `--body-file` 优先于 `--html-file` 和 `--body`
- 若存在附件，消息会构造成 multipart

## `test`

```bash
./zephyr-mail test
```

- 使用当前 SMTP 配置做连通性测试
- 会发送一封测试邮件到配置的 SMTP 用户地址
